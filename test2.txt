1.删除本地a分支
  git branch -d <a>
2.删除远程a分支
  git branch -r -d <origin/a>
3.查看分支所属
  git branch -vv
4.查看commit未push到远程分支的信息
  git cherry -v
5.查看详细的a的commit未push到远程分支a的信息
  git log <a> ^orign<a>
6.将当前分支推送到origin主机的对应分支(如果当前分支只有一个追踪分支，那么主机名都可以省略)
  git push origin
7.如果当前分支与多个主机存在追踪关系，那么这个时候-u选项会指定一个默认主机，这样后面就可以不加任何参数使用git push
  git push -u origin master
8.upstream 设置config push 默认为upstream（上游）《新版git》
  git config --global push.default upstream
tracking 设置config push 默认为tracking（追踪）《旧版git》
  git config --global push.default tracking
