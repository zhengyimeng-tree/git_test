1.将a分支关联到b分支
 git branch --set-upstream-to=origin/<b> <a>

2.将项目fetch 下来 并rebase
 git pull --rebase (fetch:拉取代码; rebase:打补丁)

3.删除本地a分支
  git branch -d <a>

4.删除远程a分支
  git push origin -d <a>

5.查看分支所属
  git branch -vv

6.查看commit未push到远程分支的信息
  git cherry -v

7.查看详细的a的commit未push到远程分支a的信息
  git log <a> ^orign<a>

8.将当前分支推送到origin主机的对应分支(如果当前分支只有一个追踪分支，那么主机名都可以省略)
  git push origin

9.如果当前分支与多个主机存在追踪关系，那么这个时候-u选项会指定一个默认主机，这样后面就可以不加任何参数使用git push
  git push -u origin master

10.upstream 设置config push 默认为upstream（上游）《新版git》
  git config --global push.default upstream

11.tracking 设置config push 默认为tracking（追踪）《旧版git》
  git config --global push.default tracking

12.把删除的状态返回
  shift + 
13.知道commit id的话，查看某次commit的操作
  git show commit-id

14.想要查看某次commit的某个文件进行了哪些修改
  git show commit-id filename

15.git stash 隐藏和恢复
  git stash pop 恢复最新的进度到工作区。git默认会把工作区和暂存区的改动都恢复到工作区。
  git stash pop --index 恢复最新的进度到工作区和暂存区。（尝试将原来暂存区的改动还恢复到暂存区）
  git stash pop stash@{1}恢复指定的进度到工作区。stash_id是通过git stash list命令得到的 
  通过git stash pop命令恢复进度后，会删除当前进度
  git stash apply [Cindex] [stash_id]
  除了不删除恢复的进度之外，其余和git stash pop 命令一样。
  git stash drop [stash_id]
  删除一个存储的进度。如果不指定stash_id，则默认删除最新的存储进度。
  git stash clear
  删除所有存储的进度。

  就用git remote prune origin 来进行修剪
